# 內容設定欄位建議（Schema 草案）

目標：把「寫作風格/TA/獨家觀點/交付形式」從口頭需求，落地成可配置欄位，讓生成更穩定、可控、可被 QA。

本文件提供：
- 建議 UI 欄位（可直接做設定頁）
- 建議後端 JSON 結構（可存 DB 或設定檔）

---

## 一、建議 UI 欄位（最小可行 + 高價值）

### 1) 文章目標
- 主要關鍵字（必填）
- 次要關鍵字/延伸關鍵字（選填，最多 10 個）
- 文章目的（必填，單選）：
  - 教學/指南
  - 比較/選購
  - 問題排解
  - 觀點/評論
- 讀完要做的事（必填，1 句）：例如「完成一份清單」「做出選擇」「開始執行步驟」。

### 2) 目標讀者（TA）
- TA 程度（必填）：新手 / 中階 / 進階
- TA 角色（選填）：上班族、家長、店主、學生…
- TA 情境（必填，1–3 句）：他為何搜尋？目前卡在哪？
- TA 禁忌/敏感點（選填）：例如「不要恐嚇式用語」「不要推銷口吻」。

### 3) 作者設定（Voice）
- 作者身分（必填）：例如「資深產品經理」「人資顧問」。
- 作者價值觀（必填，最多 3 條）：例如「務實」「以人為本」。
- 口吻（必填）：白話/專業/偏嚴謹/偏輕鬆
- 禁止句型（選填）：例如「在本文中」「讓我們一起」。

### 4) 內容交付與結構
- 交付形式（必填，至少 1 個）：
  - 步驟
  - 清單
  - 比較表
  - 範例
- 必含小節（選填）：例如「常見錯誤」「FAQ」「工具/資源」。
- 章節數量目標（選填）：例如 H2 5–7 個。
- 內容深度（必填，單選）：MVP / 標準 / 深入

### 5) 獨家觀點與案例
- 獨家觀點（選填，最多 5 條）：例如「決策框架：先排除再最佳化」。
- 可用案例（選填）：僅提供真實可用的情境；不允許 AI 虛構。
- 常見誤區（選填，最多 5 條）：用於反例與提醒。

### 6) 可信度（資料來源）
- 是否強制要求來源（必填）：是/否
- 最少來源數（可依領域預設）：
  - finance ≥ 2
  - health ≥ 1
  - 其他 ≥ 0（或 1）
- 允許的來源類型（選填）：政府/學術/交易所/監管機構/品牌官網

## 二、建議後端設定 JSON（範例）

```json
{
  "keyword": "新手投資理財入門",
  "secondaryKeywords": ["資產配置", "ETF", "風險承受度"],
  "purpose": "guide",
  "desiredAction": "建立自己的投資目標與資產配置初稿",

  "targetAudience": {
    "level": "beginner",
    "persona": "上班族",
    "scenario": "剛開始存到第一筆可運用資金，想知道如何開始投資並避免踩雷。",
    "taboos": ["恐嚇式用語", "推銷口吻"]
  },

  "author": {
    "identity": "務實派理財教練",
    "values": ["風險優先", "可持續", "不追短線"],

  },

  "deliverables": {
    "mustInclude": ["steps", "checklist"],
    "optionalSections": ["common_mistakes", "faq"],
    "targetH2Count": {"min": 5, "max": 7},
    "depth": "standard"
  },

  "originality": {
    "uniqueAngles": ["先定風險承受度再選工具", "先做『不做清單』再做『要做清單』"],
    "allowedCaseNotes": "僅可使用使用者提供的真實案例，不得虛構",

  }
}
```

## 三、QA 對應（建議）

- 產前：缺必填欄位 → 直接拒絕生成或回提示補齊。
- 產後：用 backend/docs/CONTENT_EVALUATION_STANDARDS.md 的 100 分量表做驗收。
- 產後（自動）：把「最少來源數」「交付形式」「H2 數量」「關鍵字出現」對應到 deterministic/heuristic gate。
